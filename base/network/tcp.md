# TCP
### 一、三次握手
- ACK(确认)、SYN(建立联机)是TCP标志位
- 客户端发送一个SYN=1，seq=x包，请求建立连接
- 服务端收到后发送SYN=1，ACK=1，seq=y，ack=x+1的包
- 客户端收到后发送一个ACK=1，seq=x+1，ack=y+1的的包
- 为什么要三次：理论上是4次，23步合成一步了
  - 客户端：我要建立连接
  - 服务端：我要建立连接
  - 服务端：我准备好建立连接
  - 客户端：我准备好建立连接

### 二、四次挥手
- 客户端：我要断开连接
- 服务端：我要断开连接
- 中间有个等待时间，确保包传送完成
- 服务端：我准备好断开连接
- 客户端：我准备好断开连接
- 为什么要四次：2、3步骤中间有个等待时间所以无法合并
